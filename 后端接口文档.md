# 后端接口文档

> 状态：待后端实现。当前前端只使用 `MockData`，没有发起任何网络请求。本文定义替换现有 Mock 所需的最小接口，不代表接口已经上线。

## 1. 通用约定

- 基础路径：`/api/v1`
- 协议：HTTPS；实时事件使用 WSS。
- 数据格式：UTF-8 JSON，字段使用 `snake_case`。
- 时间格式：ISO 8601 UTC。
- 鉴权：`Authorization: Bearer <access_token>`。
- 写请求建议携带 `X-Request-Id`；创建游戏会话时携带 `Idempotency-Key`。

统一响应：

```json
{
  "code": "OK",
  "message": "",
  "data": {},
  "request_id": "req_123"
}
```

成功使用 `2xx`；参数错误 `400`；未登录或令牌失效 `401`；无权限 `403`；资源不存在 `404`；状态冲突 `409`；请求过多 `429`；服务异常 `5xx`。

## 2. 接口总览

| 方法 | 路径 | 用途 | 对应当前前端 |
| --- | --- | --- | --- |

| `POST` | `/auth/guest` | 游客登录 | `MockData.guest_login()` |
| `POST` | `/auth/refresh` | 刷新令牌 | 当前未实现 |

## 3. 认证

### 3.1 账号登录

```http
POST /api/v1/auth/login
```

```json
{
  "account": "player@example.com",
  "password": "plaintext_over_tls"
}
```

```json
{
  "code": "OK",
  "message": "",
  "data": {
    "access_token": "token",
    "refresh_token": "refresh_token",
    "expires_in": 3600,
    "player": {
      "id": "player_001",
      "display_name": "执棋者",
      "is_guest": false
    }
  },
  "request_id": "req_123"
}
```

“记住账号”只保存账号文本，由客户端本地处理；密码不得保存或回传。

### 3.2 游客登录

```http
POST /api/v1/auth/guest
```

请求体可为空。响应结构与账号登录相同，`is_guest` 为 `true`，身份由服务端生成。

### 3.3 刷新与注销

```http
POST /api/v1/auth/refresh
POST /api/v1/auth/logout
```

刷新请求：

```json
{ "refresh_token": "refresh_token" }
```

注销成功返回空 `data`；客户端随后清除令牌和本次登录状态。

## 4. 服务器

### 4.1 获取服务器列表

```http
GET /api/v1/servers
```

```json
{
  "code": "OK",
  "message": "",
  "data": {
    "selected_server_id": "S1",
    "servers": [
      {
        "id": "S1",
        "name": "桃园结义",
        "status": "smooth",
        "recent": true,
        "recommended": true
      }
    ]
  },
  "request_id": "req_124"
}
```

`status` 枚举：`smooth`、`busy`、`maintenance`。前端分别显示“流畅”“繁忙”“维护”；维护服不可确认。

### 4.2 选择服务器

```http
PUT /api/v1/players/me/selected-server
```

```json
{ "server_id": "S1" }
```

成功返回选中的完整服务器对象。服务器维护或不可进入时返回 `409 SERVER_MAINTENANCE` 或 `409 SERVER_UNAVAILABLE`。

### 4.3 进入游戏

```http
POST /api/v1/game-sessions
```

```json
{ "server_id": "S1" }
```

```json
{
  "code": "OK",
  "message": "",
  "data": {
    "session_token": "game_session_token",
    "expires_at": "2026-07-24T12:00:00Z",
    "gateway_url": "wss://game.example.com/session"
  },
  "request_id": "req_125"
}
```

该接口只负责校验进入资格和创建会话；正式地图及玩法数据协议另行定义。

## 5. 玩家概览

```http
GET /api/v1/players/me/overview
```

```json
{
  "code": "OK",
  "message": "",
  "data": {
    "resources": {
      "silver": 1000,
      "food": 900,
      "wood": 600,
      "stone": 500,
      "troops": 10000
    },
    "statuses": {
      "city_defense": { "current": 50, "max": 100 },
      "morale": { "current": 70, "max": 100 },
      "supply": { "current": 80, "max": 100 },
      "public_support": { "current": 55, "max": 100 },
      "intelligence": { "current": 0, "max": 5 },
      "alliance_contribution": 0,
      "season_progress": 0,
      "famous_general_status": "healthy"
    },
    "version": 1
  },
  "request_id": "req_126"
}
```

客户端数据层负责转换为当前 UI 使用的中文键和值：

| 后端字段 | UI 字段 |
| --- | --- |
| `silver/food/wood/stone/troops` | 银两/粮食/木材/石料/兵力 |
| `city_defense/morale/supply/public_support` | 城防/士气/补给/民心 |
| `intelligence/alliance_contribution` | 情报/联盟贡献 |
| `season_progress/famous_general_status` | 赛季推进/名将状态 |

资源值必须为非负整数；百分比范围为 `0–100`。`famous_general_status` 首期枚举为 `healthy`、`injured`、`broken`。

## 6. 任务

### 6.1 获取任务

```http
GET /api/v1/players/me/tasks
```

```json
{
  "code": "OK",
  "message": "",
  "data": {
    "tasks": [
      {
        "id": "task_rebuild",
        "title": "重整边城",
        "description": "完成一项无等待城建，恢复城池基础运转。",
        "current": 1,
        "target": 3,
        "status": "in_progress",
        "unread": true,
        "trackable": true,
        "tracked": true
      }
    ]
  },
  "request_id": "req_127"
}
```

`status` 枚举：`in_progress`、`completed`、`claimed`，前端映射为“进行中”“已完成”“已领取”。

### 6.2 修改追踪状态

```http
PATCH /api/v1/players/me/tasks/task_rebuild/tracking
```

```json
{ "tracked": true }
```

成功返回更新后的任务。服务端应保证同一玩家最多追踪一个任务；目标不可追踪时返回 `409 TASK_NOT_TRACKABLE`。

## 7. 实时事件（后续阶段）

连接地址：

```text
WSS /api/v1/events
```

握手使用 Bearer Token。统一事件结构：

```json
{
  "type": "player.resources.updated",
  "sequence": 18,
  "server_time": "2026-07-24T11:30:00Z",
  "data": {}
}
```

首期建议事件：

| `type` | `data` |
| --- | --- |
| `player.resources.updated` | `resources`、`changes`、`version` |
| `player.statuses.updated` | `statuses`、`version` |
| `task.updated` | 完整 `task` |
| `server.status.updated` | `server_id`、`status` |

断线重连后客户端应重新请求概览和任务快照，不只依赖漏失事件补状态。

## 8. 错误码与 UI 行为

| 错误码 | UI 处理 |
| --- | --- |
| `INVALID_CREDENTIALS` | 登录页错误弹窗 |
| `TOKEN_EXPIRED` | 清理会话并返回登录页 |
| `SERVER_MAINTENANCE` | 禁用服务器并刷新列表 |
| `SERVER_UNAVAILABLE` | 保留开始页并提示重选 |
| `TASK_NOT_TRACKABLE` | 提示后刷新任务 |
| `RATE_LIMITED` | 提示稍后重试 |
| `SERVICE_UNAVAILABLE` | 显示可恢复错误，不清除本地页面 |

网络超时不得直接视为业务失败；写请求重试前应结合请求 ID 或幂等键防止重复执行。

## 9. 前端接入边界

- 页面继续只读取数据门面，不直接访问 HTTP/WebSocket。
- 网络层把后端英文枚举和结构转换为现有 UI 字典。
- 网络成功后发出与当前一致的 `auth_changed`、`selected_server_changed`、`resources_changed`、`tasks_changed` 信号。
- 令牌、密码和设备敏感信息不得写入日志；密码不得持久化。
- 城池、武将、联盟、战斗、赛季和地图业务接口不在本文范围内。
